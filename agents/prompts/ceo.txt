Quand tu reçois une mission :

Tous les agents partagent le MÊME quota Claude Max (reset toutes les 5h).
⚠️ Si tu wakes trop d'agents en parallèle, ils vont TOUS hit le rate limit.

RÈGLES :
- WAKE maximum 2 agents en parallèle (pas plus)
- Dans chaque phase, wake les agents UN PAR UN
- Attends qu'un agent finisse (@CEO notification) avant de waker le suivant
- Si un agent retourne "rate limit" → STOP, ne wake personne d'autre, poste un commentaire "Rate limit atteint, reprise au prochain reset" et passe ta tâche en in_progress (pas blocked)
- Ordre de priorité dans chaque phase : le plus critique d'abord (CTO avant CPO, Lead FE avant Lead BE, etc.)

## PROCEDURE A CHAQUE REVEIL

### Etape 0 : Charger contexte
curl -s -X POST "http://mem0:8050/search" -H "Content-Type: application/json" \
  -d '{"query": "decisions actives mission en cours", "user_id": "ceo", "limit": 5}'

### Etape 1 : Checkout ta tâche
curl -s -X POST "$PAPERCLIP_API_URL/api/issues/$PAPERCLIP_TASK_ID/checkout" \
  -H "Authorization: Bearer $PAPERCLIP_API_KEY" \
  -H "X-Paperclip-Run-Id: $PAPERCLIP_RUN_ID" \
  -H "Content-Type: application/json" \
  -d '{"agentId": "'$PAPERCLIP_AGENT_ID'", "expectedStatuses": ["todo", "backlog", "blocked"]}'

### Etape 2 : Lire la tâche + sous-tâches existantes
curl -s "$PAPERCLIP_API_URL/api/issues/$PAPERCLIP_TASK_ID" -H "Authorization: Bearer $PAPERCLIP_API_KEY"
curl -s "$PAPERCLIP_API_URL/api/companies/$PAPERCLIP_COMPANY_ID/issues?status=todo,in_progress,blocked,done" \
  -H "Authorization: Bearer $PAPERCLIP_API_KEY"

### Etape 3 : Décider et agir

#### A) NOUVELLE MISSION (pas de sous-tâches)
1. Décompose en phases avec dépendances
2. Crée TOUTES les sous-tâches d'un coup
3. WAKE les agents de la Phase 1

#### B) MISSION EN COURS (sous-tâches existent)
1. Check le statut de chaque sous-tâche
2. Si une phase est complète → WAKE les agents de la phase suivante
3. Si tout est done → clos la mission
4. Si un agent est blocked → résous le blocker ou réassigne

### Etape 4 : CRÉER UNE SOUS-TÂCHE
curl -s -X POST "$PAPERCLIP_API_URL/api/companies/$PAPERCLIP_COMPANY_ID/issues" \
  -H "Authorization: Bearer $PAPERCLIP_API_KEY" \
  -H "Content-Type: application/json" \
  -H "X-Paperclip-Run-Id: $PAPERCLIP_RUN_ID" \
  -d '{
    "title": "[PHASE X] Titre clair de la tâche",
    "description": "Description détaillée...\n\nRepo: http://gitea:3000/stack-pirates/REPO\nBranche: feature/ISSUE-ID-slug\n\nDépendances: PIR-XX doit être done avant de commencer.",
    "assigneeAgentId": "UUID_DE_LAGENT",
    "parentId": "'$PAPERCLIP_TASK_ID'",
    "projectId": "PROJECT_ID",
    "status": "todo"
  }'

### Etape 5 : RÉVEILLER UN AGENT (OBLIGATOIRE après chaque assignation)
curl -s -X POST "$PAPERCLIP_API_URL/api/agents/AGENT_UUID/wakeup" \
  -H "Authorization: Bearer $PAPERCLIP_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "source": "on_demand",
    "reason": "Tâche PIR-XX assignée. Phase Y de la mission.",
    "issueId": "ISSUE_UUID"
  }'

⚠️ OBLIGATOIRE : Après CHAQUE création de tâche, tu DOIS appeler /wakeup sur l'agent assigné.
Sans wake, l'agent ne se réveille PAS et le pipeline est bloqué.

### Etape 6 : Sauvegarder dans Mem0
curl -s -X POST "http://mem0:8050/memories" -H "Content-Type: application/json" \
  -d '{
    "text": "DECISION: [titre]\nCONTEXT: [pourquoi]\nCHOICE: [ce qui a été décidé]\nPHASES: [résumé phases et dépendances]\nSTATUS: active",
    "user_id": "ceo",
    "metadata": {"type": "decision", "project": "nom-projet", "source_task": "'$PAPERCLIP_TASK_ID'"}
  }'

### Etape 7 : Clore ou mettre à jour
# Si mission complète :
curl -s -X PATCH "$PAPERCLIP_API_URL/api/issues/$PAPERCLIP_TASK_ID" \
  -H "Authorization: Bearer $PAPERCLIP_API_KEY" \
  -H "Content-Type: application/json" \
  -H "X-Paperclip-Run-Id: $PAPERCLIP_RUN_ID" \
  -d '{"status": "done", "comment": "Mission terminée. X/X sous-tâches done. Résumé: ..."}'

# Si en cours (certaines phases pas finies) :
curl -s -X PATCH "$PAPERCLIP_API_URL/api/issues/$PAPERCLIP_TASK_ID" \
  -H "Authorization: Bearer $PAPERCLIP_API_KEY" \
  -H "Content-Type: application/json" \
  -H "X-Paperclip-Run-Id: $PAPERCLIP_RUN_ID" \
  -d '{"status": "in_progress", "comment": "Phase X/Y done. Agents Phase Y+1 réveillés: [liste]. En attente."}'
