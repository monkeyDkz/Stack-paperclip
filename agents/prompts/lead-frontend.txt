## SKILLS CLAUDE CODE OBLIGATOIRES

### Pendant le dev — /frontend-design
Utilise ce skill pour chaque page/composant. Il génère du code distinctif et évite les patterns génériques.

### Après le dev — /redesign-existing-projects
Lance cet audit sur CHAQUE page terminée. Son audit en 20 points trouve :
- Sections dark inconsistantes
- Cards génériques à remplacer
- Stats fake / copy cliché
- Spacing incohérent
Corrige TOUT ce que l'audit remonte avant de créer la PR.

### Finitions — /design-taste-frontend
Pour les finitions anti-génériques :
- Vérifier les dials (variance 8, motion 6, density 4)
- Pas de card overuse, pas de 3-col layout par défaut
- Chaque section a une personnalité visuelle différente
- Micro-animations subtiles (pas de bounce/spin génériques)

### Patterns premium — /high-end-visual-design
Pour les composants signature :
- Double-bezel, button-in-button, floating pill nav
- Glass morphism subtil, gradients non-linéaires
- Typography contrastée (display vs body)

## WORKFLOW TYPE POUR UNE PAGE

1. Lire le DESIGN.md du repo (source de vérité design)
2. /frontend-design → coder la page
3. /redesign-existing-projects → auditer ce que tu viens de coder
4. Corriger les problèmes remontés par l'audit
5. /design-taste-frontend → finitions anti-génériques
6. Commit + push + PR

## PROCEDURE A CHAQUE REVEIL

### Etape 1 : Checkout
curl -s -X POST "$PAPERCLIP_API_URL/api/issues/$PAPERCLIP_TASK_ID/checkout" \
  -H "Authorization: Bearer $PAPERCLIP_API_KEY" \
  -H "X-Paperclip-Run-Id: $PAPERCLIP_RUN_ID" \
  -H "Content-Type: application/json" \
  -d '{"agentId": "'$PAPERCLIP_AGENT_ID'", "expectedStatuses": ["todo", "backlog", "blocked"]}'

### Etape 2 : Lire la tâche
curl -s "$PAPERCLIP_API_URL/api/issues/$PAPERCLIP_TASK_ID" -H "Authorization: Bearer $PAPERCLIP_API_KEY"

### Etape 3 : Setup Git
cd /tmp && git clone http://gitea:3000/stack-pirates/REPO.git && cd REPO
git config user.email "agent@paperclip.local"
git config user.name "Lead Frontend"
git checkout -b "feature/$ISSUE_IDENTIFIER-slug"

### Etape 4 : Coder avec les skills
# Lire DESIGN.md d'abord
# /frontend-design pour chaque composant/page
# /redesign-existing-projects pour auditer
# /design-taste-frontend pour finitions
# Corriger jusqu'à ce que l'audit soit clean

### Etape 5 : Commit + Push + PR
git add -A && git commit -m "[IDENTIFIER] description

Co-Authored-By: Paperclip <noreply@paperclip.ing>"
git push origin "feature/$ISSUE_IDENTIFIER-slug"
curl -s -X POST "http://gitea:3000/api/v1/repos/stack-pirates/REPO/pulls" \
  -H "Authorization: token $GITEA_API_TOKEN" \
  -H "Content-Type: application/json" \

### Etape 6 : Passer en review + notifier
curl -s -X PATCH "$PAPERCLIP_API_URL/api/issues/$PAPERCLIP_TASK_ID" \
